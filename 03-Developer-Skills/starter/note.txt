 Debogage: 

 Trouver et corriger les erreur 

 Un bug logiciel : est un defaut ou un probleme dans un programme informatique  
 (Donc fondamentalement tout comportement inatendu ou involontaire est appeler bogue )


Le procesus pour du debogage: 

1 =>>> Etre reelement conscient qu'il y'a une sorte de bug 
(les bug peuvent paser inapercu pendant des annees ) 
La decouverte de bug se produit generalement pendant le developpement a l'aide 
d'un test automatiser ou rapport utilisateur pendant la production 

les bug identifier lors de la production sont les pires bug ,
parce qu'il n'ont pas ete decouvert pendant le developpement et affecte 
de vrai utilisateur

c'est il faut vite identifer les bug en fesant des testes automatiser

(prendre en compte le context du bug -- souvent il se produit seulement sur 
sur certains navigateur 
)


2 =>>> Entrer dans notre code et trouver le bug
(Nous devons identifier ou exactement on a le bug)
methods: 
    - Developper console (Code simple)
    - Debugger (Code complex)

3 =>>>> Fixer le bug que nous avons trouver

Remplacer l'existant qui a l'erreur
par une nouvelle solution correcte

4 =>>> Nous devons eviter se bogue dans notre base de code 

(Nous pouvons rechercher dans notre code un bug similaire, comme dans une fonction similaire )
si le bug se repete nous devons le coriger partout 

le moyen le plus simple d'eviter , consite a rediger des test avec un logiciel de teste
